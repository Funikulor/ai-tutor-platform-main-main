# Устранение проблем

## Network Error при входе

Если вы видите ошибку "Network Error" при попытке входа, выполните следующие шаги:

### 1. Проверьте, запущен ли бэкенд

**Windows:**
```bash
# Откройте новое окно командной строки или PowerShell
cd AdaptEd\backend
python -m uvicorn app:app --reload --port 8000
```

Или используйте:
```bash
start_backend.bat
```

**Проверка:**
- Откройте в браузере: http://localhost:8000
- Должно появиться сообщение: `{"message": "Welcome to the AdaptEd API!"}`
- Документация API: http://localhost:8000/docs

### 2. Проверьте порты

Убедитесь, что:
- **Бэкенд** работает на порту **8000**
- **Фронтенд** работает на порту **3000**
- Порты не заняты другими приложениями

### 3. Проверьте настройки API

В файле `AdaptEd/frontend/src/services/auth.ts` должен быть правильный URL:
```typescript
const API_BASE_URL = import.meta.env.VITE_API_URL || 'http://localhost:8000';
```

### 4. Очистите кэш браузера

- Нажмите `Ctrl + Shift + Delete`
- Очистите кэш и cookies
- Или используйте режим инкогнито

### 5. Проверьте консоль браузера

1. Откройте DevTools (F12)
2. Перейдите на вкладку "Console"
3. Посмотрите на ошибки
4. Перейдите на вкладку "Network"
5. Попробуйте войти снова
6. Проверьте запрос к `/auth/login` - какой статус?

## Частые проблемы

### Проблема: "CORS error"
**Решение:** Бэкенд уже настроен на разрешение всех источников. Проверьте, что бэкенд запущен.

### Проблема: "Connection refused"
**Решение:** Бэкенд не запущен. Запустите его через `start_backend.bat` или вручную.

### Проблема: "Timeout"
**Решение:** Увеличьте таймаут в `auth.ts` или проверьте, что бэкенд отвечает быстро.

### Проблема: "401 Unauthorized"
**Решение:** Неверный email или пароль. Используйте правильные учетные данные.

## Тестирование подключения

Откройте консоль браузера (F12) и выполните:

```javascript
fetch('http://localhost:8000/')
  .then(r => r.json())
  .then(console.log)
  .catch(console.error)
```

Если видите `{"message": "Welcome to the AdaptEd API!"}`, то бэкенд доступен.
