# Руководство по обновлению Frontend

Если вы обновили frontend, но изменения не отображаются на сайте, выполните следующие шаги:

## Проблема: Изменения не отображаются

### Решение 1: Перезапуск dev сервера

1. Остановите текущий dev сервер (Ctrl+C в терминале)
2. Удалите кеш:
```bash
cd AdaptEd/frontend
rm -rf node_modules/.vite
# или на Windows:
rmdir /s /q node_modules\.vite
```
3. Перезапустите:
```bash
npm run dev
```

### Решение 2: Очистка браузера

1. Откройте DevTools (F12)
2. Правый клик на кнопке обновления
3. Выберите "Очистить кеш и жесткая перезагрузка"
4. Или используйте Ctrl+Shift+R

### Решение 3: Пересборка проекта

Если используете production build:
```bash
cd AdaptEd/frontend
npm run build
```

Затем перезапустите preview:
```bash
npm run preview
```

## Проверка изменений

1. Убедитесь, что файлы сохранены
2. Проверьте консоль браузера на ошибки (F12)
3. Проверьте консоль терминала где запущен `npm run dev`
4. Убедитесь, что Vite видит изменения (должны быть сообщения о hot reload)

## Автоматическая перезагрузка

Vite должен автоматически перезагружать страницу при изменениях. Если этого не происходит:

1. Проверьте, что файл сохранен
2. Проверьте, что нет синтаксических ошибок
3. Перезапустите dev сервер

## Новые компоненты не импортируются

Если вы добавили новый компонент:

1. Проверьте правильность импорта
2. Убедитесь, что файл сохранен
3. Проверьте консоль на ошибки TypeScript/ESLint
4. Перезапустите TypeScript сервер в IDE (если используется)

## Проверка работы чата

После добавления компонента чата:

1. Убедитесь, что backend запущен на порту 8000
2. Убедитесь, что Ollama запущена (если используется)
3. Откройте вкладку "Чат с ИИ" в интерфейсе студента
4. Проверьте консоль браузера на ошибки API
